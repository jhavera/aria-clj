## Part A: Missing binary ops (math_demo)

### A1: rem in emit-binop

Add two cases to the existing `emit-binop` dispatch:

```clojure
[:rem :i32] (.visitInsn mv Opcodes/IREM)
[:rem :i64] (.visitInsn mv Opcodes/LREM)
```

### A2: cast node

`(cast i32 i64 $i)` widens an i32 value to i64. Add a new case to
`emit-expr!`:

```clojure
:cast (let [from-type (keyword (:from-type node))
            to-type   (keyword (:to-type node))]
        (emit-expr! mv ctx (:value node))
        (case [from-type to-type]
          [:i32 :i64] (.visitInsn mv Opcodes/I2L)
          [:i64 :i32] (.visitInsn mv Opcodes/L2I)
          [:i32 :f64] (.visitInsn mv Opcodes/I2D)
          [:f64 :i32] (.visitInsn mv Opcodes/D2I)
          [:i64 :f64] (.visitInsn mv Opcodes/L2D)
          [:f64 :i64] (.visitInsn mv Opcodes/D2L)
          (throw (ex-info "Unknown cast" {:from from-type :to to-type}))))
```

Read `parser.clj` to confirm the exact keys on the cast AST node before
writing this -- verify `:from-type`, `:to-type`, and `:value` are correct.

### A3: Fix print boxing for i64

The current `emit-print!` always boxes arguments as `Integer`. For i64
arguments it must box as `Long`. The fix requires knowing the type of each
print argument.

Print args in ARIA-IR have types that can be inferred from context. The
simplest approach: look up the arg's type from the locals map if it is a
`:var-ref`, otherwise inspect the node type.

Add a private helper:

```clojure
(defn- infer-expr-type
  "Best-effort type inference for a print argument.
  Returns :i64 for known i64 expressions, :i32 otherwise."
  [ctx node]
  (case (:node/type node)
    :var-ref (let [local (get-local ctx (:name node))]
               (if local (:type local) :i32))
    :int-literal :i32
    :float-literal :f64
    :bool-literal :bool
    :string-literal :str
    :i32))
```

Then in `emit-print!`, after pushing each arg value onto the stack, box
based on inferred type:

```clojure
(let [arg-type (infer-expr-type ctx arg)]
  (case arg-type
    :i64 (.visitMethodInsn mv Opcodes/INVOKESTATIC
                           "java/lang/Long" "valueOf"
                           "(J)Ljava/lang/Long;" false)
    :f64 (.visitMethodInsn mv Opcodes/INVOKESTATIC
                           "java/lang/Double" "valueOf"
                           "(D)Ljava/lang/Double;" false)
    ;; default: box as Integer
    (.visitMethodInsn mv Opcodes/INVOKESTATIC
                      "java/lang/Integer" "valueOf"
                      "(I)Ljava/lang/Integer;" false)))
```

## Part B: Pointer and memory support (bubble_sort)

This is the architectural addition in Phase 3. The JVM has no raw pointers.
The solution: model the ARIA heap as a static class with a flat int array.
Pointers become plain int indices into that array.

### B1: The AriaMemory model

```
(ptr i32)              --> :i32 at the JVM level (just an int index)
(alloc i32 N)          --> AriaMemory.alloc(N)  returns int
(load.i32 ptr)         --> AriaMemory.load(ptr) returns int
(store.i32 ptr val)    --> AriaMemory.store(ptr, val)
(free ptr)             --> AriaMemory.free(ptr) -- no-op in Phase 3
(add.i32 ptr offset)   --> regular IADD -- no special case needed
```

Pointer arithmetic (`add.i32 %arr $j`) is just integer addition. It works
automatically because pointers are already ints. No change to `emit-binop`.

### B2: Generate AriaMemory class

Add a private function parallel to `generate-intent-annotation-class`:

```clojure
(defn- generate-aria-memory-class
  "Generates bytecode for the AriaMemory runtime support class.
  This class implements the ARIA heap as a flat int array.
  Methods: alloc(int count) -> int, load(int addr) -> int,
           store(int addr, int val) -> void, free(int addr) -> void.
  Returns a byte array containing a valid .class file."
  []
  (let [cw (ClassWriter. ClassWriter/COMPUTE_FRAMES)]
    (.visit cw Opcodes/V11
            (+ Opcodes/ACC_PUBLIC Opcodes/ACC_SUPER)
            "com/ariacompiler/AriaMemory" nil "java/lang/Object" nil)

    ;; Static field: int[] heap
    (.visitField cw (+ Opcodes/ACC_PRIVATE Opcodes/ACC_STATIC)
                 "heap" "[I" nil nil)

    ;; Static field: int next (next free index)
    (.visitField cw (+ Opcodes/ACC_PRIVATE Opcodes/ACC_STATIC)
                 "next" "I" nil nil)

    ;; Static initializer: heap = new int[65536]; next = 0;
    (let [mv (.visitMethod cw Opcodes/ACC_STATIC "<clinit>" "()V" nil nil)]
      (.visitCode mv)
      (.visitLdcInsn mv (int 65536))
      (.visitIntInsn mv Opcodes/NEWARRAY Opcodes/T_INT)
      (.visitFieldInsn mv Opcodes/PUTSTATIC "com/ariacompiler/AriaMemory" "heap" "[I")
      (.visitInsn mv Opcodes/ICONST_0)
      (.visitFieldInsn mv Opcodes/PUTSTATIC "com/ariacompiler/AriaMemory" "next" "I")
      (.visitInsn mv Opcodes/RETURN)
      (.visitMaxs mv 0 0)
      (.visitEnd mv))

    ;; public static int alloc(int count)
    (let [mv (.visitMethod cw (+ Opcodes/ACC_PUBLIC Opcodes/ACC_STATIC)
                           "alloc" "(I)I" nil nil)]
      (.visitCode mv)
      ;; int addr = next;
      (.visitFieldInsn mv Opcodes/GETSTATIC "com/ariacompiler/AriaMemory" "next" "I")
      (.visitVarInsn mv Opcodes/ISTORE 1)   ; store addr
      ;; next = next + count;
      (.visitFieldInsn mv Opcodes/GETSTATIC "com/ariacompiler/AriaMemory" "next" "I")
      (.visitVarInsn mv Opcodes/ILOAD 0)    ; count param
      (.visitInsn mv Opcodes/IADD)
      (.visitFieldInsn mv Opcodes/PUTSTATIC "com/ariacompiler/AriaMemory" "next" "I")
      ;; return addr;
      (.visitVarInsn mv Opcodes/ILOAD 1)
      (.visitInsn mv Opcodes/IRETURN)
      (.visitMaxs mv 0 0)
      (.visitEnd mv))

    ;; public static int load(int addr)
    (let [mv (.visitMethod cw (+ Opcodes/ACC_PUBLIC Opcodes/ACC_STATIC)
                           "load" "(I)I" nil nil)]
      (.visitCode mv)
      (.visitFieldInsn mv Opcodes/GETSTATIC "com/ariacompiler/AriaMemory" "heap" "[I")
      (.visitVarInsn mv Opcodes/ILOAD 0)
      (.visitInsn mv Opcodes/IALOAD)
      (.visitInsn mv Opcodes/IRETURN)
      (.visitMaxs mv 0 0)
      (.visitEnd mv))

    ;; public static void store(int addr, int val)
    (let [mv (.visitMethod cw (+ Opcodes/ACC_PUBLIC Opcodes/ACC_STATIC)
                           "store" "(II)V" nil nil)]
      (.visitCode mv)
      (.visitFieldInsn mv Opcodes/GETSTATIC "com/ariacompiler/AriaMemory" "heap" "[I")
      (.visitVarInsn mv Opcodes/ILOAD 0)    ; addr
      (.visitVarInsn mv Opcodes/ILOAD 1)    ; val
      (.visitInsn mv Opcodes/IASTORE)
      (.visitInsn mv Opcodes/RETURN)
      (.visitMaxs mv 0 0)
      (.visitEnd mv))

    ;; public static void free(int addr) -- no-op
    (let [mv (.visitMethod cw (+ Opcodes/ACC_PUBLIC Opcodes/ACC_STATIC)
                           "free" "(I)V" nil nil)]
      (.visitCode mv)
      (.visitInsn mv Opcodes/RETURN)
      (.visitMaxs mv 0 0)
      (.visitEnd mv))

    (.visitEnd cw)
    (.toByteArray cw)))
```

### B3: Update aria-type-map->kw for pointer types

The current implementation throws on non-primitive types:

```clojure
(defn- aria-type-map->kw [t]
  (when t
    (if (= :primitive (:type/kind t))
      (keyword (:type/name t))
      (throw (ex-info "Non-primitive type in JVM backend" {:type t})))))
```

Pointers are represented as `:i32` at the JVM level (they are int indices).
Read `parser.clj` to confirm how pointer types are represented in the AST
before writing this. The type map for a `(ptr i32)` likely has
`{:type/kind :pointer :type/elem ...}` or similar. Update to handle it:

```clojure
(defn- aria-type-map->kw [t]
  (when t
    (case (:type/kind t)
      :primitive (keyword (:type/name t))
      :pointer   :i32   ; pointers are int indices into AriaMemory
      (throw (ex-info "Unknown type kind in JVM backend" {:type t})))))
```

Confirm the exact `:type/kind` value for pointers by reading `parser.clj`.

### B4: Add alloc, free, load, store nodes to emit-expr! and emit-stmt!

Read `parser.clj` to confirm the exact node types and keys for these before
writing. The node types are likely `:alloc`, `:free`, `:load`, `:store`.

**alloc** (expression -- returns an int address):

```clojure
:alloc (do
         (.visitLdcInsn mv (int (:count node)))
         (.visitMethodInsn mv Opcodes/INVOKESTATIC
                           "com/ariacompiler/AriaMemory"
                           "alloc" "(I)I" false))
```

**load** (expression -- reads from address, returns int):

```clojure
:load (do
        (emit-expr! mv ctx (:ptr node))
        (.visitMethodInsn mv Opcodes/INVOKESTATIC
                          "com/ariacompiler/AriaMemory"
                          "load" "(I)I" false))
```

**store** (statement -- writes to address):

```clojure
:store (do
         (emit-expr! mv ctx (:ptr node))
         (emit-expr! mv ctx (:value node))
         (.visitMethodInsn mv Opcodes/INVOKESTATIC
                           "com/ariacompiler/AriaMemory"
                           "store" "(II)V" false))
```

**free** (statement -- no-op call):

```clojure
:free (do
        (emit-expr! mv ctx (:ptr node))
        (.visitMethodInsn mv Opcodes/INVOKESTATIC
                          "com/ariacompiler/AriaMemory"
                          "free" "(I)V" false))
```

### B5: Boolean equality comparison

`(eq.bool $swapped false)` -- add `:bool` to the comparison dispatch in
`emit-i32-comparison` or handle it as a special case. Boolean equality in
JVM is just `IF_ICMPEQ` since bools are ints (0/1). The existing i32
comparison path handles it if the type suffix routes correctly. Verify
by reading the comparison emit path and tracing how `eq.bool` would arrive.
If it falls through without handling `:bool`, add it explicitly.

### Test group 10: alloc/store/load round-trip

```clojure
(deftest alloc-store-load-round-trip
  (let [ast    (alloc-store-load-fixture)  ; alloc 1 int, store 99, load it
        bytes  (codegen-jvm/generate-class ast "MemTest")
        result (load-and-invoke-with-memory bytes "MemTest" "mem_fn")]
    (is (= 99 result))))
```

`load-and-invoke-with-memory` is identical to `load-and-invoke` but also
loads the `AriaMemory` class into the same classloader:

```clojure
(defn- load-and-invoke-with-memory [class-bytes class-name method-name & args]
  (let [mem-bytes (#'aria.codegen-jvm/generate-aria-memory-class)
        loader    (proxy [ClassLoader] []
                    (findClass [name]
                      (cond
                        (= name class-name)
                        (.defineClass this name class-bytes
                                      0 (count class-bytes))
                        (= name "com.ariacompiler.AriaMemory")
                        (.defineClass this name mem-bytes
                                      0 (count mem-bytes))
                        :else
                        (throw (ClassNotFoundException. name)))))
        klass     (.loadClass loader class-name)
        method    (.getMethod klass method-name
                              (into-array Class (map type args)))]
    (.invoke method nil (into-array Object args))))
```

## What Phase 3 does NOT include

- Java standard library interop (calling java.util.*, etc.) -- future work
- Pointer types other than `(ptr i32)` -- `(ptr i64)`, `(ptr f64)` if needed
  should follow the same AriaMemory pattern but are not required by any
  current example
- GC / proper memory management -- AriaMemory uses a bump allocator with a
  65536-word heap; `free` is a no-op; this is sufficient for the examples
